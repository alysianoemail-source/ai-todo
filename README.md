# AI Todo

> 把脑子里的东西倒进去，AI 帮你拆成可执行的待办。

你是否有过这样的时刻——脑子里塞满要做的事，打开备忘录却不知道从何写起？或者面对一篇长长的学习计划、会议纪要，手动一条条提取待办，十分钟过去了还没开始干活。

AI Todo 做的就这一件事：**把混乱的输入，变成清晰可执行的任务清单。**

## 为什么不用飞书 AI / 备忘录 / Notion

| | 备忘录 / Notion | 飞书 AI | AI Todo |
|---|---|---|---|
| 输入长文 | 贴进去还是一大段字 | 对话式，生成完就沉了 | 自动提取编号、识别可行动项 |
| 分类 | 手动建文件夹 | 不分类 | AI 动态聚类，自动分组命名 |
| 优先级 | 自己判断 | 不排序 | 紧急 × 重要 × 耗时，多因子评分 |
| 子任务 | 手动缩进 | 不合并 | 相同前缀自动合并为父任务 |
| 跨平台 | 仅苹果 / 需客户端 | 需飞书 | 一个链接，浏览器即用 |
| 数据归属 | 云端（需账户） | 飞书内 | 纯本地，数据只在你的设备里 |
| 分享 | iCloud / 邀请 | 组织内 | 发个链接，对方打开就用 |

**核心差异：这不是"又一个备忘录"，而是"把你的混乱想法翻译成可执行计划的翻译器"。**

## 怎么用

### 日常速记（<100字）
一行一个 todo，点「✨ 一键更新」。AI 自动分类、打分、排序。

### 长文脑暴（≥100字）
粘贴整段文字——学习计划、会议记录、灵感碎片，甚至一段聊天记录。点「✨ 一键更新」，弹出预览面板：

- AI 提取出来的待办可以勾选/取消
- 标题点一下当场改
- 底部「+ 新增一行」随时补
- 确认后一起加入列表

### 联动飞书（可选）
- 飞书机器人实时回复：查待办、发长文提取、报告进度
- 每日定时推送提醒到手机：「你前几天说要学风控模型，现在该做了哦」
- 数据可同步到飞书文档，换设备也能恢复

## 技术细节

### AI 模型
**没有调用任何大模型 API。** 所有"智能"都是纯前端 JavaScript：

- **分类**：关键词匹配 + 动态聚类算法（基于共词分析，自动发现关联）
- **优先级**：紧急度 × 重要度 × 预估耗时的多因子评分模型
- **长文解析**：正则识别编号列表、动作动词、破折号分隔，自动清理口语填充词
- **标题优化**：去掉"最近想找时间""抽空""那个"等 20+ 种噪音词

全部在浏览器里跑，零 token 消耗，断网也能用。

### 存储
`localStorage`——浏览器自带的本地数据库。数据存在你自己的设备上：

- 你在电脑 Chrome 打开 → 数据在电脑 Chrome 里
- 你在手机 Safari 打开 → 数据在手机 Safari 里
- 清浏览器缓存数据会丢 → 定期点 📤 导出 JSON 备份
- 换设备 → 📤 旧设备导出 → 📥 新设备导入
- 连了飞书 → 自动备份到飞书文档，换设备也能恢复

### 架构
一个 HTML 文件，零依赖，零构建。

```
index.html （~1500行）
├── CSS     界面 + 响应式 + 移动端适配
├── JS      AI引擎 + 长文解析 + 数据管理
├── PWA     manifest.json + sw.js（添加到桌面）
└── 飞书    bot-listener.js（实时消息 + 定时提醒）
```

## 部署

项目就是一个静态网页，三种方式都能跑：

```bash
# 方式1: 浏览器直接打开
open index.html

# 方式2: 本地服务器
python -m http.server 8765

# 方式3: 部署到任意静态托管（Vercel / GitHub Pages / CloudStudio）
```
